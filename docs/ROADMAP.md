# Roadmap

The first public release is centred on trustworthy local annotation: safe document
imports, durable browser storage, portable editable HTML, flattened output, accessible
core interaction, and reproducible browser tests.

## Before the first public release

- Complete hands-on Chrome, Edge, Firefox, and Safari checks.
- Complete the keyboard, VoiceOver/screen-reader, 200% zoom, contrast, touch, and
  reduced-motion checklist.
- Create the public repository, enable the prepared Pages deployment, and smoke-test
  the exact deployed commit.

## Likely next improvements

- Multi-page PNG workflows and more precise vector resize/endpoint handles.
- Better stylus input, including pressure and coalesced pointer events.
- Deliberate annotation-overflow behaviour and richer document-scale performance tests.
- Optional standards-based annotation interchange.

## Research, not commitments

- Destructive redaction, only with unusually clear safety boundaries.
- Optional Read Closer reading templates and evidence/claim workflows.

Features that upload document content, add telemetry, or introduce a remote processor
are outside the default architecture and require an explicit privacy and product review.
